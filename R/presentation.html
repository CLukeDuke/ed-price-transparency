Feature: ZIP code + distance filter

The problem: All 7 hospitals always show equally — but a patient in Brandon has no reason to consider South Tampa hospitals first.

The solution:

  1. User enters a ZIP code
  2. App looks up coordinates from a Hillsborough County ZIP table
  3. Haversine formula calculates straight-line distance to each hospital
  4. A slider filters out hospitals beyond a chosen radius
haversine_distance <- function(lat1, lng1, lat2, lng2) {
  R    <- 3958.8  # Earth radius in miles
  phi1 <- lat1 * pi / 180
  phi2 <- lat2 * pi / 180
  dphi <- (lat2 - lat1) * pi / 180
  dlng <- (lng2 - lng1) * pi / 180
  a    <- sin(dphi/2)^2 +
          cos(phi1) * cos(phi2) * sin(dlng/2)^2
  2 * R * asin(sqrt(a))
}

Result: Entering ZIP 33511 (Brandon) and filtering to 15 miles automatically surfaces the two nearest hospitals — no manual sorting needed.

Feature: Insurance payer filter

The problem: “Cash price” is only useful for the uninsured. Most patients have insurance — but what does their plan actually pay?

The solution: - Dropdown with 8 Florida insurers: Florida Blue, Aetna, UnitedHealthcare, Cigna, Humana, Medicaid, Medicare - get_payer_price() applies payer-specific multipliers derived from de-identified MRF negotiated rate ranges - Prices update instantly across all cards, value boxes, and charts

get_payer_price <- function(base_cash, hosp_id, payer_id) {
  if (payer_id == "cash") return(base_cash)
  multiplier <- PAYER_MULTIPLIERS[[hosp_id]][[payer_id]]
  round(base_cash * multiplier)
}

What you see:

Medicare rates are 30-35% below cash prices

UnitedHealthcare rates are 45-55% above cash prices

The difference in what insurers pay at Tampa General for a chest pain workup:

  • Medicare: ~$2,800
  • Cash: $4,118
  • UnitedHealthcare: ~$6,200

Feature: Out-of-pocket calculator

The most actionable feature — estimates what a patient actually pays

Inputs:

  • Insurance plan selection
  • Annual deductible ($)
  • Amount already met toward deductible ($)
  • Coinsurance percentage after deductible (%)
  • Out-of-pocket maximum ($)

The calculation:

deduct_remaining <- max(0, deductible - deductible_met)

# Step 1: apply remaining deductible
deduct_applied <- min(base_price, deduct_remaining)
after_deduct   <- base_price - deduct_applied

# Step 2: apply coinsurance to remainder
coins_applied  <- after_deduct * coinsurance

# Step 3: cap at out-of-pocket maximum
total_oop  <- deduct_applied + coins_applied
oop_spent  <- deductible_met * coinsurance
final_oop  <- min(total_oop, max(0, oop_max - oop_spent))

Example: $3,000 deductible, $500 met, 20% coinsurance, $6,000 OOP max → chest pain at Tampa General costs you $2,963 not $4,118

Feature: Treatment definitions

The problem: Patients see CPT 84484 — Troponin I and have no idea what it means or why they’re being charged for it.

Every billing code badge and service line item now has a hover tooltip with three parts:

  1. What it is — full name and CPT code
  2. What it does — plain-language description
  3. Why you see this — why it appears on an ED bill specifically

Example: Troponin I

Measures a protein released into the bloodstream when heart muscle cells are damaged. Elevated levels strongly indicate a heart attack. Results typically return in 1–2 hours and are often repeated after 3–6 hours.

Why you see this: The definitive blood test for ruling a heart attack in or out. Cannot be skipped for a serious chest pain workup — you may be billed for it twice if a repeat draw is ordered.

Design decisions and trade-offs

What we chose and why:

  • Static price data + live Claude API — avoids 2GB downloads on every search while still enabling unknown symptom mapping
  • Payer multipliers instead of raw MRF payer data — full MRF payer extraction requires parsing hundreds of plan name variations per hospital; multipliers give clinically reasonable estimates immediately
  • Haversine over Google Maps API — no API key required, accurate enough for within-county comparisons
  • Quarto for presentation, Shiny for product — right tool for each job

What we would improve with more time:

  • Parse actual payer-specific negotiated rates from MRFs by plan name
  • Add real-time wait time data from hospital websites
  • Geocode user address (not just ZIP centroid) for more accurate distances
  • Add urgent care comparison — many conditions are cheaper at urgent care
  • Deploy to shinyapps.io for persistent public access
  • Add a shareable URL encoding current search state

The average American cannot answer this question:

“If I go to the nearest ER with chest pain, how much will I pay?”
